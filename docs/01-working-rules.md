## Purpose
This document defines the working rules for this project to ensure disciplined execution, structured thinking, and consistent learning. The goal is to develop real engineering habits, not just complete the project.

## Working Rules

1. No blind AI coding  
All features must be attempted and reasoned through before asking AI for help.

2. Explain before code  
Before implementing any feature, clearly define:
- problem
- inputs and outputs
- business rules
- edge cases

3. Think first, then verify  
Write my own solution or approach first, then use AI to review and refine.

4. One issue at a time  
Work on one defined task (issue) at a time. Do not jump between tasks.

5. Slow is acceptable, correctness is required  
Prioritize understanding and correctness over speed.

6. Document meaningful decisions  
Any important design or implementation decision must be recorded.

7. Consistency over intensity  
Even small progress is acceptable as long as it is consistent.

8. 我们可以慢，可以错，但必须坚持下去，我们才能学得更多。

## Coding Principles

1. Code must be understandable  
Write code that can be explained clearly, not just code that works.

2. Validate assumptions  
Do not assume logic is correct — verify with examples and edge cases.

3. Handle edge cases explicitly  
Always consider invalid inputs, limits, and unexpected scenarios.

4. Keep functions simple and focused  
Each function should have a clear single responsibility.

5. Review before asking for help  
Always review and debug my own code before involving AI.

6. Learn from mistakes  
When something is wrong, understand why before fixing it.

7. Avoid unnecessary complexity  
Do not introduce advanced patterns or tools unless there is a clear need.

## AI Usage Rules

1. AI is used for guidance, not implementation  
Do not copy full solutions without understanding.

2. Always attempt first  
Write my own approach or code before asking AI.

3. Ask for hints, not answers  
Prefer partial guidance over full solutions.

4. Review AI output critically  
Do not trust AI blindly — verify logic and correctness.

5. Use AI for:
- explanations
- debugging hints
- architecture discussion

6. Avoid using AI for:
- generating full features without understanding
- bypassing thinking process
