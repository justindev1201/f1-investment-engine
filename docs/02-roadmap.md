# Roadmap

## Rationale
The purpose of this roadmap is to define high-level phases to guide structured development and ensure steady progress towards a deployable MVP.

## Phases

### Phase 1: Project Setup & Definition
Goal: Establish a strong foundation for development

- Set up repository, documentation, and project workflow
- Define project scope, rules, and roadmap
- Decide initial tech stack and architecture direction
- Prepare development environment

---

### Phase 2: Core Backend Development
Goal: Build the core game logic and system functionality

- Implement user registration and login
- Implement room creation and joining via share code
- Implement round lifecycle (open → lock → result → payout)
- Implement investment validation and pool cap enforcement
- Implement payout calculation and jackpot logic
- Ensure correctness through basic testing and validation

---

### Phase 3: System Completion & Basic Interface
Goal: Make the system usable end-to-end

- Build a simple frontend interface for user interaction
- Connect frontend to backend APIs
- Allow users to complete a full game round through the UI
- Display results and updated balances clearly

---

### Phase 4: Deployment & User Testing
Goal: Make the system accessible and test with real users

- Deploy application to a live environment
- Conduct small group testing with target users
- Identify and fix critical bugs
- Improve usability based on feedback

---

### Phase 5: Stabilization & Improvement
Goal: Improve system reliability and prepare for extended use

- Refine UI and user experience
- Improve error handling and edge case coverage
- Optimize performance where necessary
- Prepare for additional features beyond MVP
